XMLEDITOR.NET(xml.exe)について

【概要】
　複雑なXML文書でも簡単、安全、確実に編集ができるハイブリッドビュー型XMLエディタです。
　
【特徴】
　XML文書をテキストエディタで編集・閲覧する際、以下のような不満を持つことはありませんか？
　・開始タグや終了タグといったXML文書特有の記述が面倒
　・作成したXML文書が整形式かどうか、DTDと整合性が取れているかいちいちチェックするのが面倒
　・タグ等のXML特有のマークアップ情報が邪魔で読みづらい
　・タグや属性といった文書構造に拠った検索・置換をおこないたい

　XMLEDITOR.NETは以下の特徴により、XML文書の編集・閲覧作業を効率化します。
　・XML文書をツリー形式や表形式といった、より見やすい形式で表示します。
　・ツリー形式や表形式のビューにより、XML文書の記述方法に詳しくない人でも簡単に編集できます。
　・IE相当の整形・色分けされた状態でXMLソースを直接編集できます。
　・常に整形式の文書が生成されるため、意識してチェックする必要性や手戻りがありません。
　・文書に変更を加える都度、構文チェック（整形式、DTD、XMLSchema整合性）を実行し、手戻りを最小限に抑えます。
　・XSL変換をサポート。
　・XML文書に特化した検索機能により、ノード型、名前、値、XPathを指定した検索が可能です。
　・XML文書専用Grep機能を搭載。出力フォーマットを自由に指定して、複数のXML文書から抽出した情報を出力することが可能です。
　
　その他の特徴
　・レジストリフリー。レジストリを使用しません。（関連付けを設定した場合を除きます）
　
【対応OS】
　Windows 98/Me/NT/2000/XP/2003/Vista

【事前にインストールが必要なライブラリ】
　.NET Framework 2.0再頒布パッケージ　※

　　※入手先 : http://www.microsoft.com/japan/msdn/netframework/downloads/

【インストール方法】
　　１．アーカイブファイル（XmlEditor.zip）を適当なフォルダに解凍してください。

【アンインストール方法】
　　１．拡張子の関連付けを設定している場合、設定画面で関連付けを解除してください。
　　２．起動中のXMLEDITOR.NETを全て終了してください。
　　３．XMLEDITOR.NETフォルダを削除してください。

【既知の問題】
　・ソースビューの表示で太字を使用するオプションをONにした場合記号表示がずれる。
　・印刷プレビューダイアログから印刷を実行すると白紙が印刷されてしまう。
　・XHTML1.1のDTDを読み込むと名前空間"xmlns="http://www.w3.org/1999/xhtml"がすべてのタグに設定されてしまう。
　　（この現象は設定で「外部参照を解決する」「妥当性検証」をオフにすることにより取りあえず回避できます。
　　　ただしそうすると自動的に妥当性検証がおこなわれなくなるので必要に応じてツールから手動で妥当性検証を実行してください。）
